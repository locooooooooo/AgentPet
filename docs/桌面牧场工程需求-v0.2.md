## 1. 范围与非范围

### 1.1 In Scope（v0.2 实施期）

- 牧场窗口：独立 BrowserWindow，**默认桌面 widget 形态（透明穿透 + 常驻）+ 可切浮动模式**
- 牧场窗口视觉 = **圈养地**（有围栏边界的"小牧场"，不是无边草原）
- 8 动物渲染：每只一个可识别形象 + 状态动作 + 状态 quote
- 状态通知：状态变更触发 3 通道（牧场内通知带 / 系统通知 / 控制舱红点）
- 召唤控制舱：双击动物 + 牧场右键菜单
- 性格档：chatty / quiet / silent，影响通知密度
- 自动靠边：拖到屏幕边缘停留后吸附（仅 floating 模式生效）
- 窗口位置 / 尺寸 / 模式持久化

### 1.2 Out of Scope（v0.2 不做）

明确不做 —— 不在判定条件中：

- Live2D / Spine / 真骨骼动画
- 二次元形象 / 养成游戏化形态
- 伴侣云端同步 / 账号系统
- 付费 / 高级版
- 语音输入 / 输出
- 自定义形象编辑器
- 默认启用真实 agent 接入（与 `docs/orchestration/connectors.json` 策略一致）
- 引入 Tailwind（沿用手写 CSS，与现有项目风格一致）

## 3. 数据模型

### 3.1 沿用类型（不引入新顶层字段）

`AgentSnapshot` 维持现状：

```typescript
interface AgentSnapshot {
  version: 1;
  updatedAt: string;
  agents: AIAgent[];      // 8 个
  runtime: Record<string, NiuMaRuntimeState>;   // 每个 agent 一个 runtime state
  messages: AgentSystemMessage[];                // 系统消息流（通知带用）
```

**所有现有字段就是牧场动画 + 通知带的全部数据源**。不引入 `mood`、`xp`、`relationship` 等养成型字段。

### 4.2 主进程扩展

`electron/main.ts` 新增：

```typescript
let ranchWindow: BrowserWindow | null = null;
let ranchPrefs: RanchPrefs;

function loadRanchPrefs(): RanchPrefs { /* IO ranch-prefs.json */ }
function saveRanchPrefs(): void { /* write file */ }

function createRanchWindow(): void { /* 类 createPetWindow，但按 RanchPrefs 初始化 */ }

function registerRanchIpc(): void {
  ipcMain.handle('ranch:get-prefs', () => ranchPrefs);
  ipcMain.handle('ranch:set-prefs', (_e, patch) => {
    ranchPrefs = { ...ranchPrefs, ...patch };
    saveRanchPrefs();
    ranchWindow?.webContents.send('ranch:prefs-changed', ranchPrefs);
    return ranchPrefs;
  });
  ipcMain.handle('ranch:open-cockpit', () => {
    mainWindow?.show(); mainWindow?.focus();
  });
  ipcMain.handle('ranch:set-bounds', (_e, b) => {
    ranchWindow?.setBounds(b);
    ranchPrefs = { ...ranchPrefs, position: { x: b.x, y: b.y }, size: { width: b.width, height: b.height } };
    saveRanchPrefs();
  });
  ipcMain.handle('ranch:toggle-mode', () => {
    const next: RanchMode = ranchPrefs.mode === 'floating' ? 'sleeping' : 'floating';
    applyRanchMode(next);
    return next;
  });
  ipcMain.handle('ranch:select-agent', (_e, agentId) => {
    ranchPrefs = { ...ranchPrefs, selectedAgentId: agentId };
    saveRanchPrefs();
  });
  ipcMain.on('ranch:right-click', (_e, menu) => {
    /* build electron Menu from items */
  });
  ipcMain.on('ranch:request-notify', (_e, payload) => {
    if (!ranchPrefs.notifyPrefs.system) return;
    new Notification({ title: payload.title, body: payload.body }).show();
  });
}

function applyRanchMode(mode: RanchMode): void {
  if (!ranchWindow) return;
  if (mode === 'floating') {
    ranchWindow.setAlwaysOnTop(true, 'floating');
    ranchWindow.setIgnoreMouseEvents(false);
  } else {
    // desktop mode（默认）—— 桌面 widget
    ranchWindow.setAlwaysOnTop(true, 'screen-saver');
    ranchWindow.setIgnoreMouseEvents({ forward: true });  // 圈外穿透到桌面
  }
  ranchPrefs = { ...ranchPrefs, mode };
  saveRanchPrefs();
}
```

## 5. IPC API 一览

### 5.1 已有（零改动）

| 通道 | 方向 | Payload | 返回 |
| --- | --- | --- | --- |
| `agents:get-snapshot` | R→M | – | `AgentSnapshot` |
| `agents:reset-seed` | R→M | – | `AgentSnapshot` |
| `agents:create-task` | R→M | `CreateTaskInput` | `AgentSnapshot` |
| `agents:stop-task` | R→M | `{ agentId, taskId }` | `AgentSnapshot` |
| `agents:clear-completed-tasks` | R→M | `{ agentId }` | `AgentSnapshot` |
| `agents:apply-niuma-action` | R→M | `{ agentId, action }` | `AgentSnapshot` |
| `agents:snapshot-changed` | M→R event | – | `AgentSnapshot` |

### 5.2 新增（v0.2）

| 通道 | 方向 | Payload | 返回 | 说明 |
| --- | --- | --- | --- | --- |
| `ranch:get-prefs` | R→M | – | `RanchPrefs` | 读当前偏好 |
| `ranch:set-prefs` | R→M | `Partial<RanchPrefs>` | `RanchPrefs` | 改偏好并持久化；广播 `ranch:prefs-changed` |
| `ranch:open-cockpit` | R→M | – | – | 召唤控制舱 |
| `ranch:set-bounds` | R→M | `{ x,y,width,height }` | – | 写入位置 + 持久化 |
| `ranch:toggle-mode` | R→M | – | `RanchMode` | desktop/floating 切换 |
| `ranch:select-agent` | R→M | `agentId: string` | – | 写入 selectedAgentId |
| `ranch:right-click` | R→M | `MenuItem[]` | – | 用 Electron `Menu.buildFromTemplate` 弹出 |
| `ranch:request-notify` | R→M | `{ title, body }` | – | 主进程发系统通知 |
| `ranch:prefs-changed` | M→R event | – | `RanchPrefs` | 偏好广播 |

**契约约束**：
- 所有 invoke 通道**必须返回**成功值或抛 Error；前端 `await` 必须处理 reject
- 所有 broadcast 通道必须保证 race-safe：先后到达顺序在主进程内 FIFO
- `ranch:request-notify` 受 `notifyPrefs.system` 控制；主进程在通道层就拒绝，不依赖 renderer 判断

## 6. 功能需求 FR

> 每条 FR 锁定一个独立可交付单位。**实施顺序建议** 按编号；并发实施需明确写时分工。

### FR-001 牧场窗口生命周期

- **触发**: 应用启动 / 托盘菜单"召唤桌面牧场"
- **入口**: 主进程 `createRanchWindow()`
- **行为**:
  - 创建 `BrowserWindow`：`frame:false, transparent:true, focusable:false（默认 mode=desktop）, skipTaskbar:true, show:false`
  - `setAlwaysOnTop(true, 'screen-saver')`（**默认 desktop widget 层级**）
  - `setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true })`
  - 尺寸默认 `{ width: 640, height: 360 }`（**Q3** 默认已重定：圈养矩形；待拍可改）
  - 位置默认屏幕右下角 80px 偏移（**Q4** 默认已重定：widget 习惯；待拍可改）
  - **默认 `setIgnoreMouseEvents({ forward: true })`** —— 圈外透明穿透到桌面
  - 加载 `ranch.html`
  - `ready-to-show` 后显示（**永远不 hide**；desktop widget 模式）
  - 用户"关闭"行为保留映射为：floating 模式才生效；desktop 模式永远不被关
- **验收**:
  ```
  npm run lint   # 0 错
  npm run build  # 通过
  启动后：牧场窗口永远可见，控制舱最小化不影响牧场可见
  默认 mode=desktop：圈外鼠标点击穿透到下面应用
  ```

### FR-002 8 动物渲染与身份识别

- **输入**: `AgentSnapshot.agents`（8 个）
- **数据源**: `src/ranch/data/agentAnimals.ts`（8 个 `AgentDef`）
- **位置默认**: 8 个动物按 `RanchLayout` 分配"工位"
- **行为**:
  - 渲染 8 只动物到 `RanchCanvas`
  - 每只有 `displayName` + `codename`，hover 显示
  - 鼠标 hover 高亮 + 选中圈
  - 点击 → 选中 + 出现 `SelectedOverlay`
- **Q 影响**: `Q1` 视觉风格决定 `visual.kind` 是 emoji / pixel / svg；`Q2` 决定是否摆脱"4 牛 4 马"
- **验收**:
  - 启动后 8 只动物可见，每只可点中
  - hover 一只 → 高亮 + 头顶显示名字 + codename

### FR-003 状态 ↔ 动画剧本

- **输入**: 任意 agent 的 `runtime.status`（14 种 NiuMaStatus 之一）
- **数据源**: `src/ranch/data/statusActions.ts`（14 → AnimalAction 映射）
- **行为**:
  - `pickAnimalAction(agent, runtime, salt)` 返回对应 `AnimalAction`
  - 动物在 `<300ms` 内切换到对应 `pose`
  - 同一状态随机盐保证动画不重复
- **状态 → 剧本对应**:

  | NiuMaStatus | pose | motion | bubble 备注 |
  | --- | --- | --- | --- |
  | idle | rest_sleep | idle | quote 当 bubble |
  | coding | work_type | shake | 「码起来了」 |
  | debugging | work_pace | walk (绕圈) | – |
  | meeting | meeting_circle | walk (到中央广场) | – |
  | coffee | coffee_sip | idle | – |
  | testing | test_drip | shake | – |
  | deploying | deploy_pray | idle | 远景火箭动效 |
  | done | done_cheer | cheer | 「完工」 |
  | overtime | overnight_dream | lay | 月亮飘上（背景） |
  | drinkingWater | water_drink | walk (到饮水区) | – |
  | panicking | panic_smoke | shake | spark=smoke |
  | slacking | slack_phone | idle | – |
  | praying | praying | idle | particle=spark |
  | demanding | demand_angry | shake | 💢 |

- **验收**:
  - 在控制舱给 codex 下任务，`sleep → work_type → done_cheer` 切换连贯
  - 切换延迟 < 300ms（人工秒表或 video 估时）

### FR-004 状态通知带

- **位置**: 牧场画布顶部水平条
- **数据源**: 订阅 `agents:snapshot-changed`，对比 `messages[0]` 与上次值
- **行为**:
  - 新消息出现 → 顶部弹 1 条 toast，3s 收回
  - 缩略显示：`title + content`，按 `personality` 决定密度
  - chatty: 所有 info / success / warning 都弹
  - quiet:  只 success / warning / error
  - silent: 不弹气泡，仅任务统计数字变化
- **验收**:
  - 控制舱下发任务后，牧场通知带在 <500ms 内弹出至少一条 toast
  - 切到 silent 后通知带不弹（但任务状态依然刷新）

### FR-005 召唤控制舱

- **触发**:
  - 鼠标 hover 牧场任意动物上方，**双击该动物**
  - 牧场窗口内任意位置，**右键 → "召唤控制舱"**
- **行为**: `ranchWindow` 内 `ranch:open-cockpit` IPC → 主进程 `mainWindow.show() + focus()`
- **Q 影响**: `Q5`（召唤姿势）确认入参
- **验收**:
  - 双击任意动物 → 牧场窗口不消失，控制舱窗口出现并抢焦点
  - 右键牧场空白区 → 出现菜单（含"召唤控制舱"项）→ 点击后同上

### FR-006 系统通知通道

- **触发**:
  - 任务 status `running → success` 或 `running → error` 时
  - 主进程 IPC handler `rancho:request-notify` 主动推送
- **行为**:
  - 检查 `ranchPrefs.notifyPrefs.system` → false 不弹
  - 调用 `new Notification({ title, body, icon: 'icon/codex.png' 用 agent accent }).show()`
  - 通知点击后把控制舱调到前台 + 选中该 agent（可选，后续再加）
- **验收**:
  - 控制舱给 codex 下 simulated 任务，等其跑完 → OS 收到通知 toast
  - 在控制舱 / 牧场设置里关掉 system 后，下发任务不再弹系统通知

### FR-007 性格档

- **存储**: `ranch-prefs.json` 的 `personality` 字段
- **入口**: 控制舱设置页（v0.2 控制舱内增加"桌面牧场"设置区）+ 牧场右键菜单"切性格档"
- **三档**:
  - chatty: 通知带频繁，气泡随状态都弹
  - quiet: 只在状态切换点弹
  - silent: 通知带关闭，仅动画变化
- **Q 影响**: `Q8`（性格档 v0.2 必做?）确认
- **验收**:
  - 切换 chatty ↔ silent 后，下发任务 → 通知带密度按预期变化

### FR-008 桌面 widget 模式（默认） vs 浮动模式

> **拍板来源**：用户 2026-07-01 明确"做成牛马圈养的形态，常驻桌面，并且是无边框的形式"。
>
> **本节取代此前"漂浮 / 栖息"二元设计**。原"栖息"概念并入 `desktop` 模式（= 桌面 widget 行为），原"漂浮"改为 `floating`（= 显式浮窗行为）。

- **存储**: `ranchPrefs.mode: 'desktop' | 'floating'`
- **默认**: `'desktop'`（**默认**；v0.2 装好后就是这个模式）

#### FR-008.1 desktop 模式（默认）

- **窗口设置**:
  - `BrowserWindow({ transparent: true, frame: false, focusable: false, skipTaskbar: true })`
  - `setAlwaysOnTop(true, 'screen-saver')` —— Win / macOS 的"桌面级"层级（在桌面壁纸之上、应用之下）
  - `setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true })` —— 跨虚拟桌面
  - `setIgnoreMouseEvents({ forward: true })` —— **圈外区域不响应鼠标，让点击穿透到下面应用**
- **触发区处理**（renderer 端）:
  - 用 `<Animal />` 组件包裹的 DOM 节点：`pointer-events: auto`（可点击）
  - 围栏、背景装饰、文字：`pointer-events: none`（穿透到桌面）
  - 进入/离开动物区域时，IPC 通知主进程 toggle `setIgnoreMouseEvents` —— 但 v0.2 简化版可以**全局默认穿透，仅动物区域不穿透**（不动态切换，更省 IPC）
- **行为**:
  - 永远可见 —— `app.quit()` 不影响；`hide()` 也不调用
  - 不抢焦点 → 写代码时不会被牧场干扰
  - 鼠标 hover 动物 → 动物抬头 + 弹出气泡
  - 鼠标点击动物 → 选中 / 双击召唤控制舱
- **验收**:
  - 牧场启动后**永远最可见**，即便控制舱 / 其他应用最大化
  - 鼠标点击牧场窗口的"围栏空白处" → 落到下面的应用（**穿透**）
  - 鼠标点击 animal 区域 → 响应点击

#### FR-008.2 floating 模式（高级 / 显式切换）

- **窗口设置**:
  - `setAlwaysOnTop(true, 'floating')`
  - `focusable: true`
  - 不开 setIgnoreMouseEvents
- **行为**:
  - 牧场窗口**抢焦点**，整体响应鼠标
  - 与 desktop 模式相比：圈外区域也可点（拖、悬浮）
  - 适合用户**主动管理**牧场的场景（编辑位置、改性格档）
- **验收**:
  - 切到 floating → 牧场整体可拖、不会"穿过"
  - 切回 desktop → 圈外再次"回到桌面"

#### FR-008.3 切换入口

- 牧场右键菜单"切为浮动模式 / 切为桌面 widget"
- 控制舱"桌面牧场设置"区
- 写 `ranchPrefs.mode` 持久化
- 不通过 IPC 强制重启 BrowserWindow：仅 `setAlwaysOnTop / setIgnoreMouseEvents` 两个 API 切换

### FR-009 自动靠边

- **触发**: 拖拽牧场窗口靠近屏幕边缘 + 停留 1 秒
- **行为**:
  - 计算最近的 vertical / horizontal 边缘（屏幕 workArea）
  - `setBounds` 贴边
  - 写 `ranchPrefs.dockedEdge`
- **Q 影响**: 与 FR-001 位置默认值协同
- **验收**:
  - 拖牧场到右边缘 → 松开 → 1s 内吸附到右边缘
  - 拖离边缘 → 不吸附

### FR-010 控制舱未读红点

- **触发**: 牧场窗口 `desktop` 模式下被遮蔽 / 不可见期间，仍有 `messages[0]` 变更
- **行为**:
  - 主进程维护一个"未读计数"，每次 `agents:snapshot-changed` 触发 +1（仅当 ranchWindow 不可见）
  - 控制舱窗口 title 改为 `桌面牧场 · 控制舱 (N)`
  - 红点用 emoji 或 favicon 表达（v0.2 用 emoji 足够）
- **验收**:
  - 切到 desktop + 牧场被遮蔽 → 下任务 → 控制舱显示 `(1)`
  - 牧场回到可见区 → 计数归零

### FR-011 圈养形态视觉（围栏 + 圈地）

> **拍板来源**：用户 2026-07-01"做成牛马圈养的形态"。
> 这是与桌面 widget 模式绑定的视觉规格。

- **强制约束**:
  - 牧场窗口必须有**明确的"圈养地"边界**（不是无边草原）
  - 圈外区域视觉上是"完全透明"，**渗透下面桌面应用**
  - 视觉上像"桌面上一块小型牧场 / 牛圈"，而不是"满屏草原"
- **围栏视觉**:
  - 用 **栅栏 SVG / box-shadow border 双线**模仿围栏
  - 颜色沿用 `--border` + `--green`（既有色板，§6.5.2 约束）
  - 围栏内壁 4 角可放"木桩"（装饰元素，agent accent ring 染色）
  - 围栏高 8~12px，外侧延伸 4~8px 的**视觉过渡阴影**（增强"嵌入式"感）
- **圈内地形**:
  - 默认 5 个分区（与 v0.2 FR-002 同）
  - 分区之间用极浅虚线（`border: 1px dashed rgba(255,255,255,0.05)`）暗示
- **8 动物行为约束**:
  - 动物**不出圈**（默认 position 限定在圈内地形）
  - 动物移动 / 走到饮水区 → 路径点都在圈内
  - 动物 panic_smoke / done_cheer 时，**圈外不冒泡**（粒子限定在圈内）
- **尺寸**:
  - 默认 `640 × 360`（横长矩形围栏，**比之前的 720×320 略矮**）
  - 最小 `320 × 200`（紧凑 widget）
  - 最大 `1200 × 500`（横展）
- **拖拽 / 缩放**:
  - 整圈可拖（drag handle 设为围栏边框）
  - 右下角放"缩放手柄"（小三角 / 点）
  - 拖离屏幕边缘 → 不自动靠边（避免 widget 跳动）
- **验收**:
  - 牧场窗口有清晰"围栏"边界
  - 鼠标点圈外 5px → 落到下面应用（穿透）
  - 鼠标点圈内 → 围栏 / 动物响应
  - 缩放到 320×200 + 放到 1200×500 → 围栏视觉保持一致

### 6.5.2 必须继承的视觉语言（隐性契约）

桌面牧场**新建**的所有 CSS / 组件必须**沿用**如下体系（不强求 token 名一致，但风格一致）：

| 类目 | 既有体系 | 桌面牧场怎么用 |
| --- | --- | --- |
| 色板 | `--green #00d68f / --blue #4da3ff / --orange #ff7a1a / --red #ff4d5f` | 牧场通知带 / 状态色点 |
| 背景层 | `--panel rgba(17,17,17,.88)` / `--border rgba(255,255,255,.09)` | 牧场分组块 |
| accent ring | `color-mix(in srgb, var(--agent-accent) 70%, white 10%)` + 雾化 box-shadow | 8 动物外圈、选中态 |
| 表情 meta | `(•_•) 💤` ASCII + 表情 行 | 牧场通知带的 status 文字 |
| emoji 字体 | 系统 emoji + `drop-shadow` | 8 动物本体 |
| 动画曲线 | `shake 520ms ease-in-out infinite` / `breathe 3.6s ease-in-out infinite` / `blink 4s ease-in-out infinite` | 动物动效（HUD 通知）|

> **关键原则**：桌面牧场绝不**重新发明**一套色板 / 动画曲线 / emoji 体系。即便实现需要新 keyframes，也要复用 `ease-in-out`、`infinite` 节奏并写出"引用自 `src/index.css` 的等价曲线"，不引入**审美漂移**。

### 6.5.3 隐含的 Q1 拍板（视觉风格 = A 档）

- ❌ SVG 几何拟人 → 新建视觉语言，**违反 §6.5.2**

> 这里的"摆脱 4 牛 4 马"Q2 仍待用户拍（A 档下摆脱 vs 保留的取舍不变）。

### 6.5.4 增量新增的样式必填字段

新建 CSS 必须在 `src/ranch/styles/ranch.css` 内，**不写全局选择器**：不污染 `body / * / :root`（除非通过 `:where(#ranch-root)` 或类似 scoping）。

| ① task card 信息量等价 | 截图上半 orchestration 区 | 5 张 card 同权重，**用户不知道看哪张** |
| ② connectors / status 两排并列 | 截图中部 | 6 chip + 3 connector card 语义切碎，**视觉无主次** |
| ③ 右侧详情面板堆叠 | 截图右侧 30% | 8 块信息无视觉断点，**成"一堵墙"** |
| ④ 中央区 4 卡 + 1 浮窗 | 截图中央 | 视觉干扰，可能在开发中加的悬浮枢纽未在代码找到 |

> 桌面牧场的 UI 不能重蹈这 4 个覆辙。

### 6.7.2 桌面牧场的 3 级信息层级（**不超过 3 级**）

| 层级 | 内容 | 视觉权重 | 出现时机 | 占比 |
| --- | --- | --- | --- | --- |
| **L1 主视觉** | 8 动物 emoji + accent ring | 60~80px + accent 发亮 | 永远在 | 占窗口 70~80% |
| **L2 次视觉** | 1 条状态气泡（agent 短名 + 1 句话 + 时间）| 12~14px + 灰文字 + accent 描边 | 状态变更 1.5s 临时出现 | 占窗口 10~15% |
| **L3 入口** | "召唤控制舱"图标 + "设置"图标 | 24px 极小图标 | 鼠标 hover 牧场时显形 | 占窗口 5% |

**禁止出现的视觉**：
- ❌ 彩色背景渐变 / 草地 / 远山 / 服务器机柜等"游戏场景"
- ❌ 进度条 / 大字标语 / 多行卡片
- ❌ "体力 / CPU 温度"等详细 metric
- ❌ 任务队列 / 命令输入框 / "立即下发"按钮
- ❌ orchestration 5 张 task card / connector policy
- ❌ 第 4 级层级（L0"全空"、L4"详细面板"）

### 6.7.3 桌面牧场**绝对不做**的 6 件事（基于截图根因）

| 不做 | 理由（对应截图根因）|
| --- | --- |
| ❌ 不放任务队列 | 避免 ③ 详情面板堆叠 |
| ❌ 不放命令输入框 | 同上 |
| ❌ 不放详细 metric | 避免 ③ "一堵墙" |
| ❌ 不放 orchestration task card | 避免 ① 信息量等价 |
| ❌ 不放 connector policy | 避免 ② 两排并列 |
| ❌ 不放"立即下发"按钮 | 桌面牧场只发**通知**，不直接**操作**（操作归控制舱）|

### 6.7.4 围栏的"嵌入"视觉（避免显式游戏场景）

| 项 | 规则 |
| --- | --- |
| 围栏边框 | 1px hairline，**`var(--border)` 色（沿用既有）**，**不要** accent 色描边 |
| 围栏阴影 | 极弱 0 1px 3px **内阴影**（不是外阴影），让圈"嵌入"桌面 |
| 围栏 4 角 | 4 个小圆点（直径 6px），accent 色，**仅在 hover 时显形**（默认隐形）|
| 内部背景 | **完全透明**（desktop mode + 圈外穿透是已有 FR-008.1，圈内也保持透明）|
| 远景 / 草地 | **不画**（避免变成"牧场物语"风格）|
| 工位区域 | 用 0.5px hairline dashed 暗示（极弱），**不画实体地板** |
| 顶部 / 底部 HUD | 透明，无背景色 |

> 围栏要"嵌入"桌面，像"桌面上一片被圈起来的地"，不是"画了一个牧场背景图"。

### 6.7.5 8 动物的"主视觉"严格规范

| 项 | 规则 | 反例 |
| --- | --- | --- |
| emoji 字符 | 沿用 `agent.avatar`（**逐字符同源 §6.6.3**）| ❌ 改用 sprite / SVG |
| 字符尺寸 | 56~72px | ❌ 缩到 32px（不显眼）/ 拉到 96px（挤）|
| accent ring | `var(--agent-accent)` + 0 0 12px agent-accent 雾化 | ❌ 改用 hard border / box-shadow 实线 |
| 表情 meta 行 | `(•_•) 💤` 这种 `STATE_METAS.expression` | ❌ 改成"工作"/"睡觉" 等中文 |
| 状态文字 | `STATE_METAS.name` 中文名（**最多 8 个字**）| ❌ 加英文 / 加感叹号 / 加 emoji |
| 体力 / 温度条 | **不放**（L1 极简）| ❌ 复制控制舱的 Meter 组件 |
| 头像 / 名字 / 描述 | 不放 | ❌ 放 8 段说明文字 |

### 6.7.6 状态气泡的"次视觉"严格规范

只允许一种气泡：

```
┌─────────────────────────────────┐
│ 🐂 codex · 开干                │
│ 任务「定位构建失败」  · 1.5s 前 │
└─────────────────────────────────┘
```

- 位置：底部中央，靠 L1 区域下方 8px
- 字体：12~14px sans-serif
- 背景：`rgba(0, 0, 0, 0.6)` + 4px hairline `var(--border)`
- 圆角：6px
- 进出：fade-in 200ms / fade-out 400ms
- 出现时机：snapshot.messages 变化（**L1 区域上方 1 条**，不堆叠）
- 自动收回：1.5s 后 fade-out
- 排队：相同 agent 1s 内多条消息**合并**（不堆叠）

> 气泡只 1 条 / 1.5s 临时，绝**不**长驻。

### 6.7.7 入口（L3）的"小图标"规范

| 图标 | 行为 | 出现条件 |
| --- | --- | --- |
| 召唤控制舱 | IPC `ranch:open-cockpit` | 鼠标 hover 牧场时显形 |
| 设置 | 切 ranch prefs | 鼠标 hover 牧场时显形 |
| 切模式 | desktop ↔ floating | 鼠标 hover 牧场时显形 |

- 位置：右下角 8px 边距，**横排 3 个**，每个 24px
- 颜色：`var(--text-muted)`，hover 变 `var(--agent-accent)`（**取当前 active agent 的 accent**）
- 默认**完全不显示**（鼠标不在牧场时）
- 没有"标签文字"（纯图标）

## 9. 数据契约

### 9.1 RanchPrefs IO

主进程：

```typescript
function loadRanchPrefs(): RanchPrefs {
  const path = ranchPrefsPath();
  if (!fs.existsSync(path)) {
    const seed = createRanchSeedPrefs();
    fs.writeFileSync(path, JSON.stringify(seed, null, 2), 'utf8');
    return seed;
  }
  try {
    const parsed = JSON.parse(fs.readFileSync(path, 'utf8')) as unknown;
    return normalizeRanchPrefs(parsed);
  } catch {
    return createRanchSeedPrefs();
  }
}
```
