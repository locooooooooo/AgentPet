# 🐄 真实生产环境状态打通与牛马表情联动方案

本方案旨在解决“非假动画演示”的核心诉求，通过打通 Electron 主进程、本地进程管道（Connector Gate）与前端状态机，实现 **Codex 等 Agent 在真实工作时的表情与状态实时联动**。

---

## 一、 状态映射与表情联动规则

当 Agent 接入真实生产环境后，其状态不再由前端定时器模拟，而是由**真实任务的生命周期与执行进度**决定：

| 真实任务状态 (Task Lifecycle) | 对应牛马状态 (NiuMaStatus) | 视觉表情 (Visual Expression) | 粒子特效 (Particles) | 行为表现 (Behavior) |
| :--- | :--- | :--- | :--- | :--- |
| **未开始/空闲** | `idle` / 默认性格态 | `(•_•) 💤` 或性格默认表情 | 无 | 熟睡或在格子间内轻微呼吸晃动 |
| **任务刚下发 (0% - 10%)** | `coding` (疯狂码砖) | `(╬☉д⊙) ⌨️` | 🔥 粒子 | 键盘冒烟，Emoji 快速左右抖动 |
| **任务执行中 (10% - 90%)** | 随机忙碌态 (见下文) | 随机切换 | 随机 | 随机播放忙碌动画，避免单一动作枯燥 |
| **任务部署/发布中** | `deploying` (拜佛发布) | `(🙏_🙏) 🚀` | 🚀 粒子 | 3D 旋转，向生产环境祈祷 |
| **任务成功完成 (100%)** | `done` (按时下班) | `(^_^) ✅` | 🌸 鲜花粒子 | 向上跳跃，欢呼雀跃，气泡：“收工！” |
| **任务执行失败 (Error)** | `panicking` (生产救火) | `(╯°□°)╯ 🔥` | 🚨 警报 + 💀 骷髅 | 剧烈震动，格子间闪烁红色警报光晕 |

### 💡 正在执行中的“随机忙碌”机制
当任务处于 `running` 状态且进度在 `10% - 90%` 之间时，状态机每隔 5 秒在以下忙碌状态中随机切换，以表现真实的“格子间忙碌生态”：
*   `coding` (疯狂码砖) $\rightarrow$ 键盘冒烟
*   `debugging` (揪发排障) $\rightarrow$ 拿着放大镜在工位踱步
*   `testing` (压测跑路) $\rightarrow$ 试管滴答，Emoji 震颤
*   `demanding` (怒怼需求) $\rightarrow$ 拍桌子，冒烟粒子

## 二、 技术实现架构：如何打通真实状态

要实现真实状态打通，必须通过 **Electron IPC 管道** 实时监听本地进程的输出与退出状态：

```
┌────────────────────────────────────────────────────────┐
│                  本地真实进程 (Node.js)                 │
│  (执行 npm run build / git commit / python 脚本等)     │
└───────────────────────────┬────────────────────────────┘
                            │ stdout / stderr 管道流
                            ▼
┌────────────────────────────────────────────────────────┐
│                 Electron 主进程 (Main)                 │
│  - 监听 child_process 的 spawn 实例                    │
│  - 实时捕获 exitCode (0=成功, 非0=失败)                 │
└───────────────────────────┬────────────────────────────┘
                            │ IPC 通信 (onSnapshotChanged)
                            ▼
┌────────────────────────────────────────────────────────┐
│                 前端渲染进程 (React UI)                 │
│  - 接收最新 Snapshot，解析任务 status ('success'/'error')│
│  - 驱动 NiuMaAvatar 播放对应的帧动画与粒子特效          │
└────────────────────────────────────────────────────────┘
```

### 1. 进程状态捕获 (主进程核心逻辑)
在 Electron 主进程中，通过 `child_process.spawn` 启动真实任务，并绑定事件监听：
```typescript
const child = spawn(command, args, { cwd });

// 1. 捕获流式日志，证明“正在拉磨”
child.stdout.on('data', (data) => {
  sendLogsToFrontend(taskId, data.toString());
});

// 2. 监听进程结束，打通“成功/失败”表情
child.on('close', (code) => {
  if (code === 0) {
    // 任务成功 -> 触发 done 状态与鲜花粒子
    updateAgentTaskStatus(agentId, taskId, 'success');
  } else {
    // 任务失败 -> 触发 panicking 状态与警报粒子
    updateAgentTaskStatus(agentId, taskId, 'error');
  }
});
```

### 2. 前端状态机接管 (React 状态映射)
在 `getNiuMaEffectiveStatus` 中，直接根据**真实任务的运行状态**映射牛马表情，不再使用纯时间模拟：
```typescript
export function getNiuMaEffectiveStatus(agent: AIAgent, runtime: NiuMaRuntimeState): NiuMaStatus {
  const runningTask = agent.tasks.find((task) => task.status === 'running');
  
  // 1. 如果有正在运行的真实任务
  if (runningTask) {
    if (runtime.isFueledByPie) return 'overtime'; // 被画饼超频
    
    // 刚开始 (0-10%) 必定在疯狂码砖
    if (runningTask.progress < 10) return 'coding';
    
    // 部署中
    if (runningTask.command.includes('deploy') || runningTask.command.includes('release')) {
      return 'deploying';
    }
    
    // 10% - 90% 之间，如果 customState 是忙碌态之一，则保留；否则随机分配一个忙碌态
    const busyStates: NiuMaStatus[] = ['coding', 'debugging', 'testing', 'demanding'];
    if (runtime.customState && busyStates.includes(runtime.customState)) {
      return runtime.customState;
    }
    return busyStates[Math.floor(runningTask.progress % busyStates.length)] ?? 'coding';
  }

  // 2. 如果最新完成的任务是失败的，呈现崩溃救火态
  const latestTask = agent.tasks[0];
  if (latestTask && latestTask.status === 'error') {
    return 'panicking';
  }
  
  // 3. 如果最新完成的任务是成功的，呈现下班欢呼态
  if (latestTask && latestTask.status === 'success') {
    // 成功状态保留一段时间后回到默认态
    const timeSinceEnd = Date.now() - new Date(latestTask.endTime || '').getTime();
    if (timeSinceEnd < 10000) { // 10秒内保持欢呼
      return 'done';
    }
  }

  // 4. 无任务时，回归默认性格态或用户手动切的摸鱼态
  return runtime.customState ?? AGENT_PERSONALITY[agent.id] ?? 'idle';
}
```
