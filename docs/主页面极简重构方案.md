> **收口日期**：2026-07-03
> **收口原因**：本方案方向（单栏聚焦 + 悬浮抽屉 + 真实状态打通）与"视觉降噪"方向一致，且完整覆盖了 v0.3 §〇·quinary §末"控制舱本身的'不简洁'问题"4 个根因，被合并进 v3.0 的 P0 阶段执行。

本方案围绕**“极简视觉、情绪共鸣、真实打通、无负担陪伴”**的核心卖点进行深度迭代。通过无情降噪，将界面重构为一款兼具**生产力**与**解压陪伴**的极简桌面控制舱。

---

## 一、 核心卖点与设计哲学 (Core Philosophy)

1.  **情绪价值最大化**：保留并优化“牛马 Emoji 动画”与“黑话台词”，让用户在枯燥的开发中获得会心一笑的解压感。
2.  **视觉噪音最小化**：砍掉所有非必要的开发调试文本，将三栏布局重构为**“单栏聚焦矩阵”**，让牛马工位成为视觉的绝对焦点。
3.  **真实状态打通**：牛马的表情与动作**100% 绑定本地真实进程的生命周期**，拒绝虚假的演示动画。
4.  **无负担陪伴**：界面干净、克制，平时作为桌面挂件无缝融入工作流，需要时一键唤出。

---

## 二、 极简视觉与布局重构 (Minimalist Layout)

彻底废除原有的三栏拥挤布局，重构为**“单栏聚焦 + 悬浮抽屉”**的极简结构：

```
┌────────────────────────────────────────────────────────────────────────┐
│                      Header (仅保留 Logo、极简 KPI)                    │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│                         Center Stage (1fr)                             │
│                         [ 干净的牛马工位矩阵 ]                         │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

### 1. 彻底移除视觉噪音 (De-cluttering)
*   **砍掉左侧轨道**：彻底删除“调度总览”、“角色分工”、“Connector管控”等大段调试文本。
*   **砍掉中央顶部面板**：删除冗余的 `Primary Worksurface` 面板，避免信息重复展示。
*   **Header 瘦身**：删除开发备注文案，高度从 120px 压缩至 60px，释放垂直空间。

### 2. 极简牛马卡片 (Clean Workstation Card)
*   **默认状态（极简）**：卡片仅展示 **Emoji 动画、牛马名称、极简状态标签**。
*   **悬停显露 (Hover to Reveal)**：
    *   **指标淡入**：“脑门发热度 (CPU)”与“剩余电量 (Energy)”指示条默认隐藏，鼠标悬停时才淡入显示。
    *   **操作栏浮现**：卡片底部的 5 个操作按钮（画饼、咖啡、鞭策、摸鱼、派活）默认隐藏，悬停时以半透明悬浮条形式浮现。

### 3. 隐藏式控制台 (Drawer Console)
*   右侧控制台默认完全隐藏。
*   **唤醒机制**：点击任意牛马卡片，控制台以平滑的 `spring` 动画从右侧滑出；点击空白处自动收起，确保平时界面 100% 干净。

## 三、 真实生产环境状态打通 (Real-world Integration)

牛马的表情与动作不再由前端定时器模拟，而是由**真实任务的生命周期**决定：

*   **未开始/空闲** $\rightarrow$ `idle` (熟睡躺平) 或性格默认态。
*   **任务刚下发 (0% - 10%)** $\rightarrow$ `coding` (疯狂码砖)，Emoji 快速左右抖动，伴随 🔥 粒子。
*   **任务执行中 (10% - 90%)** $\rightarrow$ **随机忙碌态**。每隔 5 秒在 `coding` (码砖)、`debugging` (排障)、`testing` (压测)、`demanding` (怼需求) 之间随机切换，表现真实的“格子间忙碌生态”。
*   **任务成功完成 (100%)** $\rightarrow$ `done` (按时下班)，Emoji 向上跳跃，伴随 🌸 鲜花粒子，持续 10 秒后回归默认态。
*   **任务执行失败 (Error)** $\rightarrow$ `panicking` (生产救火)，Emoji 剧烈震动，格子间闪烁红色警报光晕（🚨），直到下一次任务下发。

---

## 四、 克制的微交互 (Restrained Micro-interactions)

1.  **平滑渐变切态**：状态切换采用平滑的淡入淡出（Fade）与微小的缩放（Scale），去除刺眼的剧烈抖动。
2.  **优雅的气泡台词**：气泡对话框采用极简的半透明毛玻璃效果，出现时缓缓淡入，3秒后自动优雅淡出。
3.  **轻量级流光**：状态管道的流光改为极细、半透明的线条微光，像呼吸一样时隐时现，绝不刺眼。
