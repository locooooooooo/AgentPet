### 为什么是这个名字（用户拍板理由复盘 + 制作人补注）

## 〇·ter 桌面 widget 圈养形态（已拍板）

> **用户拍板**（2026-07-01）："我们做成牛马圈养的形态，能不能做成常驻桌面，并且是无边框的形式？"

**三句话定位**：
- **圈养形态**：牧场就是桌面上一块**圈了围栏的小牧场**，不是无边草原
- **常驻桌面**：牧场永远在最可见层，不会被其他应用完全遮住
- **无边框**：窗口没有标题栏；圈外区域**透明且鼠标可穿透到桌面**

**对原设计的推翻/强化**：

| 原 v0.2 | v0.3 拍板后 |
| --- | --- |
| 模式 `floating \| sleeping` | **`desktop \| floating`**，**desktop 为默认** |
| 视觉隐喻"草原" | **强化为"圈养地"**（明确边界 + 围栏 + 8 动物不出圈）|
| 默认尺寸 720×320 | **640×360（横长矩形围栏）** |
| 默认位置 屏幕底部中央 | **屏幕右下角 80px 偏移**（widget 习惯位置）|
| 透明区域不可点击是默认 | **圈外透明 + 鼠标穿透到桌面**，**圈内可点** |

**典型使用形态**：
> 用户打开电脑，桌面右下角有一个 640×360 的小牧场，里面 8 只牛马各自干活/打盹。鼠标移到桌面上其它位置完全感受不到牧场的存在，但鼠标移进牧场圈，围栏/动物立刻响应。牧场永远在，控制舱被最小化/关闭也不影响。控制舱被召唤出来时，牧场**不抢焦点**，控制舱照常用。

> 详细工程规格（FR-008 / FR-011）见 `docs/桌面牧场工程需求-v0.2.md` §FR-008 / §FR-011

## 〇·quarter 产品卖点保护（战略强制约束）

> **用户拍板**（2026-07-01）："我们始终要保留中间的牛马的 uiux 动效，这个是我们整个产品的卖点。"

**整个产品的卖点 = 控制舱中央 4×2 的 8 张 agent card**，由以下资产共同构成：

| 资产 | 文件 | 卖点作用 |
| --- | --- | --- |
| emoji 字符体系（🐂/🐎）| `agent.avatar` 字段 + `icon/*.png/.jpeg` | 视觉锚点 |
| accent ring 染色 | `var(--agent-accent)` CSS 变量 | 强辨识 |
| 14 套 `STATE_METAS` | `agentCore.ts` | 情绪脚本 |
| shake / breathe / blink keyframes | `index.css` `1158-1169` 行 | 活物感 |
| NiuMaAvatar 组件 | `NiuMaAvatar.tsx` 35 行 | 形象原子 |
| 4×2 网格 + slot-label + agent-card-hit | `NiuMaWorkspace.tsx` 134-237 行 | 信息密度 |
| `is-hot/coffee/cool/done` 切换视觉 | `index.css` 593-607 行 | 状态戏剧性 |

**这 8 张卡 = 这个产品的签名**。截图、分享、对外传播每次都靠它。

**任何改动这一处的方案 = 砍掉产品灵魂**。这一处不是"还改不改"的自由度问题。

- 桌面牧场的 8 动物 = 控制舱中央 8 卡的**同源可缩放投影**
- emoji 字符、accent ring、表情脚本、keyframes 函数**逐字符一致**
- 桌面牧场不能"独立"成为另一个产品 —— 它必须依附卖点

## 〇·bis UI 风格继承（已拍板硬约束）

> **用户拍板**（2026-07-01）："当前 UI 动画我都做好了，不要破坏这个 UI 风格跟素材。"

桌面牧场**不是新建一套 UI 产品**，是**控制舱视觉的桌面延伸**：

### 不可修改的既有资产清单

任何 v0.3 实施**严禁修改**以下文件 / 资产：

- `src/components/NiuMaAvatar.tsx`
- `src/components/NiuMaWorkspace.tsx`
- `src/index.css`（1219 行手写 CSS）
- `src/lib/agentCore.ts`（含 `STATE_METAS`）
- `src/lib/orchestrationStatus.ts`
- `icon/codex.png` 等 8 张（**M5 才用**：`notification icon` 来源）
- `lucide-react` 依赖
- `motion` 依赖
- `STATE_METAS` 的 `expression` 字段（如 `(╬☉д⊙) ⌨️`，已被 NiuMaAvatar 用）
- 既有动画 `@keyframes shake/breathe/blink`

### 必须继承的视觉语言

桌面牧场**新建**的所有 CSS / 组件必须**沿用**如下体系：

| 类目 | 既有体系 | 桌面牧场怎么用 |
| --- | --- | --- |
| 色板 | `--green #00d68f / --blue #4da3ff / --orange #ff7a1a / --red #ff4d5f` | 牧场通知带 / 状态色点 |
| 背景层 | `--panel rgba(17,17,17,.88)` / `--border rgba(255,255,255,.09)` | 牧场分组块 |
| accent ring | `color-mix(in srgb, var(--agent-accent) 70%, white 10%)` + 雾化 box-shadow | 8 动物外圈、选中态 |
| 表情 meta | `(•_•) 💤` ASCII + 表情行 | 牧场通知带的 status 文字 |
| emoji 字体 | 系统 emoji + `drop-shadow` | 8 动物本体 |
| 动画曲线 | `shake 520ms ease-in-out infinite` / `breathe 3.6s ease-in-out infinite` / `blink 4s ease-in-out infinite` | 动物动效（HUD 通知）|

> **关键原则**：桌面牧场绝不**重新发明**一套色板 / 动画曲线 / emoji 体系。即便实现需要新 keyframes，也要复用 `ease-in-out`、`infinite` 节奏，写出"引用自 `src/index.css` 的等价曲线"，**不引入审美漂移**。

### 增量新增的样式必填字段

新建 CSS 必须在 `src/ranch/styles/ranch.css` 内，**不写全局选择器**：不污染 `body / * / :root`（除非通过 `:where(#ranch-root)` 或类似 scoping）。

```css
/* 桌面牧场样式都包在 #ranch-root 之下 */
#ranch-root { /* 局部 token，可镜像 control cockpit 的 token */ }
#ranch-root * { /* 局部盒模型 */ }
```

> 工程实现细节见 `docs/桌面牧场工程需求-v0.2.md` §6.5

## 〇·quinary UI 简洁与 3 级信息层级（已拍板硬约束）

> **用户拍板**（2026-07-02）："帮我优化下后续制作的 UI 相关的风格，希望能够简洁看起来干净一点，然后区分重要的信息。"

桌面牧场**不是控制舱的"压缩版"**，是控制舱的"桌面延伸"：
- **控制舱** = 信息详查 + 操作面板（中央 8 卡 + orchestration + 详情面板 + 任务队列）
- **桌面牧场** = **一眼态势 + 通知终端**（8 动物 + 1 气泡 + 1 入口）

### 截图诊断：控制舱"不简洁"的 4 个根因

| 根因 | 位置 | 描述 |
| --- | --- | --- |
| ① task card 信息量等价 | orchestration 区 | 5 张 card 同权重，**用户不知道看哪张** |
| ② connectors / status 两排并列 | 状态 chip + connector card | 6 chip + 3 connector card 语义切碎，**视觉无主次** |
| ③ 右侧详情面板堆叠 | 右侧 30% 区域 | 8 块信息无视觉断点，**成"一堵墙"** |
| ④ 中央区 4 卡 + 1 浮窗 | 中央区 | 视觉干扰（看截图中央的"批发排障 / 吹水 / 欧布 / 中价广场 / 发布台"圆形枢纽）|

> 桌面牧场的 UI **不能重蹈这 4 个覆辙**。

### 桌面牧场的 3 级信息层级（**不超过 3 级**）

```
┌─────────────────────────────────────────────┐
│ 牧场窗口 640×360（圈外透明穿透）             │
│                                              │
│        🐂      🦊      🦉      🐢    ← L1 70~80%
│     （accent ring + emoji + 1 行表情）       │
│        🐎      🦨      🦥      🦓    ← L1
│                                              │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  ← 围栏 hairline
│  ┌────────────────────────────────┐         │
│  │ 🐂 codex · 开干  · 1.5s 前    │ ← L2 临时
│  └────────────────────────────────┘         │
│                              ⊙⊙⊙  ← L3 hover 显形
│                              (3 个 24px 图标)
└─────────────────────────────────────────────┘
```

| 层级 | 内容 | 视觉权重 | 出现时机 | 占比 |
| --- | --- | --- | --- | --- |
| **L1 主视觉** | 8 动物 emoji + accent ring | 60~80px + accent 发亮 | 永远在 | 70~80% |
| **L2 次视觉** | 1 条状态气泡（agent 短名 + 1 句话 + 时间）| 12~14px + 灰文字 + accent 描边 | 状态变更 1.5s 临时出现 | 10~15% |
| **L3 入口** | "召唤控制舱"图标 + "设置"图标 + "切模式"图标 | 24px 极小图标 | 鼠标 hover 牧场时显形 | 5% |

**禁止出现的视觉**：
- ❌ 彩色背景渐变 / 草地 / 远山 / 服务器机柜等"游戏场景"
- ❌ 进度条 / 大字标语 / 多行卡片
- ❌ "体力 / CPU 温度"等详细 metric
- ❌ 任务队列 / 命令输入框 / "立即下发"按钮
- ❌ orchestration 5 张 task card / connector policy
- ❌ 第 4 级层级（L0"全空"、L4"详细面板"）

### 桌面牧场**绝对不做**的 6 件事

| 不做 | 理由（对应截图根因）|
| --- | --- |
| ❌ 不放任务队列 | 避免 ③ 详情面板堆叠 |
| ❌ 不放命令输入框 | 同上 |
| ❌ 不放详细 metric | 避免 ③ "一堵墙" |
| ❌ 不放 orchestration task card | 避免 ① 信息量等价 |
| ❌ 不放 connector policy | 避免 ② 两排并列 |
| ❌ 不放"立即下发"按钮 | 桌面牧场只发**通知**，不直接**操作**（操作归控制舱）|

### 围栏的"嵌入"视觉（避免显式游戏场景）

| 项 | 规则 |
| --- | --- |
| 围栏边框 | 1px hairline，**`var(--border)` 色（沿用既有）**，**不要** accent 色描边 |
| 围栏阴影 | 极弱 0 1px 3px **内阴影**（不是外阴影），让圈"嵌入"桌面 |
| 围栏 4 角 | 4 个小圆点（直径 6px），accent 色，**仅在 hover 时显形**（默认隐形）|
| 内部背景 | **完全透明**（desktop mode + 圈外穿透是 FR-008.1，圈内也保持透明）|
| 远景 / 草地 | **不画**（避免变成"牧场物语"风格）|
| 工位区域 | 用 0.5px hairline dashed 暗示（极弱），**不画实体地板** |
| 顶部 / 底部 HUD | 透明，无背景色 |

> 围栏要"嵌入"桌面，像"桌面上一片被圈起来的地"，不是"画了一个牧场背景图"。

### 8 动物的"主视觉"严格规范

| 项 | 规则 | 反例 |
| --- | --- | --- |
| emoji 字符 | 沿用 `agent.avatar`（**逐字符同源 §〇·quarter**）| ❌ 改用 sprite / SVG |
| 字符尺寸 | 56~72px | ❌ 缩到 32px（不显眼）/ 拉到 96px（挤）|
| accent ring | `var(--agent-accent)` + 0 0 12px agent-accent 雾化 | ❌ 改用 hard border / box-shadow 实线 |
| 表情 meta 行 | `(•_•) 💤` 这种 `STATE_METAS.expression` | ❌ 改成"工作"/"睡觉" 等中文 |
| 状态文字 | `STATE_METAS.name` 中文名（**最多 8 个字**）| ❌ 加英文 / 加感叹号 / 加 emoji |
| 体力 / 温度条 | **不放**（L1 极简）| ❌ 复制控制舱的 Meter 组件 |
| 头像 / 名字 / 描述 | 不放 | ❌ 放 8 段说明文字 |

### 状态气泡的"次视觉"严格规范

只允许一种气泡：

```
┌─────────────────────────────────┐
│ 🐂 codex · 开干                │
│ 任务「定位构建失败」  · 1.5s 前 │
└─────────────────────────────────┘
```

- 位置：底部中央，靠 L1 区域下方 8px
- 字体：12~14px sans-serif
- 背景：`rgba(0, 0, 0, 0.6)` + 4px hairline `var(--border)`
- 圆角：6px
- 进出：fade-in 200ms / fade-out 400ms
- 出现时机：snapshot.messages 变化（**L1 区域上方 1 条**，不堆叠）
- 自动收回：1.5s 后 fade-out
- 排队：相同 agent 1s 内多条消息**合并**（不堆叠）

> 气泡只 1 条 / 1.5s 临时，绝**不**长驻。

### 入口（L3）的"小图标"规范

| 图标 | 行为 | 出现条件 |
| --- | --- | --- |
| 召唤控制舱 | IPC `ranch:open-cockpit` | 鼠标 hover 牧场时显形 |
| 设置 | 切 ranch prefs | 鼠标 hover 牧场时显形 |
| 切模式 | desktop ↔ floating | 鼠标 hover 牧场时显形 |

- 位置：右下角 8px 边距，**横排 3 个**，每个 24px
- 颜色：`var(--text-muted)`，hover 变 `var(--agent-accent)`（**取当前 active agent 的 accent**）
- 默认**完全不显示**（鼠标不在牧场时）
- 没有"标签文字"（纯图标）

### 控制舱本身的"不简洁"问题（v0.3+ 范围）

**本节不约束控制舱 UI**。控制舱的"信息密度优化"是 v0.3+ 的话题，且**受 §〇·quarter / §〇·bis 强约束**（中央 8 卡不能动）。

> 工程实现 + 验收命令见 `docs/桌面牧场工程需求-v0.2.md` §6.7

## 二、产品定位

### 2.1 一句话定位

> 把多 agent 系统从控制舱里的"卡片"解放到桌面的一片**真实牧场**，8 个 agent 各自以独立的动物形象在这片场地上活着、工作、抱怨、打盹。开发者抬头瞥一眼就懂全盘状态。

### 2.2 这片牧场是什么

一个**桌面 widget**（默认 desktop mode，透明穿透 + 常驻 + 圈外不抢焦点），窗口内是一片完整的小型场景画布：
- **围栏边界**：1px hairline，4 角小圆点（hover 显形）
- **L1 主视觉**：8 只动物各自活动（emoji + accent ring + STATE_METAS 表情 + keyframes）
- **L2 次视觉**：1 条状态气泡（临时 1.5s）
- **L3 入口**：3 个 24px 图标（hover 显形）

### 2.3 它不是什么

- ❌ 不是"桌面宠物 → 养成 = 主线"的养成型游戏
- ❌ 不是"桌面装饰 → 没有功能性"的小摆件
- ❌ 不是"桌面面板 → 工整严肃"的 control surface（**那是控制舱**）
- ❌ 不是控制舱的"压缩版"（**桌面牧场 ≠ 简化版控制舱**）
- ✅ 是"**功能性场景**"：状态可视化 + 通知 + 偶尔的陪伴感

### 2.4 控制舱 vs 桌面牧场 角色分工

| | 控制舱 | 桌面牧场 |
| --- | --- | --- |
| 角色 | 信息详查 + 操作面板 | **一眼态势 + 通知终端** |
| 信息密度 | 100% | **20% 以下** |
| 任务下发 | ✅ 立即下发按钮 | ❌ 不放 |
| 任务队列 | ✅ 完整列表 | ❌ 不放 |
| 详细 metric | ✅ 体力/温度/任务数 | ❌ 不放 |
| Orchestration | ✅ 5 task card | ❌ 不放 |
| 视觉层级 | 5+ 级（堆叠）| **3 级封顶** |

## 三、用户场景

### 场景 A · "瞥一眼就懂"
> 你打开电脑写代码，牧场窗口已经飘在屏幕右下角。**你不需要点开它** —— 抬头瞥一眼就能读出：**codex 正在工位前站立，爪子动得快**（coding）、**qoder 趴着酣睡**（idle）、**openclaw 站在远处摔倒冒烟**（panicking）、**hermes 在角落里打盹**（overtime）。
>
> 信息密度 = 一眼 = 全场态势感知。

**核心交付物**：8 只动物的"动作语义"必须自解释、不能歧义。

### 场景 B · 故障情感化
> 一个任务栈崩了。openclaw 在牧场里翻倒、抖一抖、头上冒烟；状态带弹一条「openclaw 生产炸了，能救就救」；系统通知同步发「openclaw 任务异常，建议查看」。
>
> 拟人化 + 多通道，而不是刷屏红字。

**核心交付物**：故障状态在牧场里有"事故现场感"，不是"日志告警"。

### 场景 C · 长跑 watchdog（安静汇报）
> 三个长任务跑起来。codex、trae、minimax 三只动物都各自在工位上码着代码；它们有自己的步态 / 节奏区分彼此身份。完成一只就在状态带冒泡「codex 完工了」。
>
> 牧场的"工作密度"直接反映你的工作负载。

**核心交付物**：8 只的差异化身份（光看就知道谁是 codex 谁是 trae）。

### 场景 D · 闲置的活物感
> 没任务时，所有动物各回各家 —— 有的趴草地、有的去饮水区、有的窝在棚里打盹。鼠标停在哪只上，它抬头看你一下，**不打扰你**。

**核心交付物**：idle 时 8 只都有"各自的位置默认"，没任务不是全员静止。

### 场景 E · 召唤控制舱
> 你想看任务详情。**右键牧场窗口**或敲场地的"门铃" → 弹出菜单 → "调出控制舱" → 控制舱从托盘弹出。
>
> 牧场只是"陪伴 + 总览"，要管理还是得去控制舱。

**核心交付物**：双形态共存，牧场不替代控制舱。

## 四、形态决策

### D1. 8 只动物完全平权

**任何一只都不是主角**。这就是 v0.2 与 v0.1 最大的区别。

- v0.1 的"主控切换"是隐喻为"切角色扮演者"
- v0.3 没有角色扮演者 —— 8 个演员都有戏份，每个是主角

### D2. 单画布 vs 多分屏

**单画布一片牧场**，所有 8 只动物在同一片草地上各占一隅。

- 单画布 = "统一场景感"（这是性格差异的载体）
- 多分屏会变成"控制舱化石"

### D3. 牧场是否分"功能区"

**轻分区** —— 不硬画地砖线，靠"地标物件"暗示（实际 v0.3 不画地砖）：

- 工位区：每个 agent 一张小桌（**不强求画桌子**，可仅靠位置暗示）
- 休息区：草堆 / 棚屋（drag / stay 状态）
- 饮水区：水盆
- 报警区：远处一根冒烟的烟囱（表示 panicking）
- 中心区：牧场中央的小广场（庆祝 / 集合）

> 实际绘制见 §〇·quinary 围栏"嵌入"视觉 —— 围栏内不做实体地板，用 0.5px hairline dashed 暗示。

### D4. 牧场是横长 vs 竖长 vs 自由

**横长（landscape）**，尺寸默认 640×360，可缩到 320×200，最大 1200×500。

### D5. 牧场是否 alwaysOnTop

**默认 desktop widget 形态**（`desktop` mode），可切 `floating` 模式（详见 §〇·ter）。

## 六、交互地图

| 触发 | 反馈 | 层级 |
| --- | --- | --- |
| 鼠标移到某只动物 | 头顶冒气泡：quote 文案 0.6s 出 + 1.5s 后收回；动物抬头 | L1 |
| 鼠标 hover 牧场任意位置 | L3 入口图标显形 | L3 |
| 单击某只动物 | 选中，主相机略推近；右侧弹出该 agent 的 mini 信息卡（任务数 / 状态条 / 简短 quote）| L1 → 临时 L4（**v0.3 简化为 1.5s 气泡**，不做 mini 卡）|
| 双击某只动物 | 召唤控制舱 + 选该 agent | L1 → L3 召唤 |
| 右键牧场任意处 | 弹出菜单：召唤控制舱 / 投喂 / 咖啡 / 摸鱼 / 切模式 / 设置 | L3 |
| 拖拽牧场窗口 | 整片场景跟着动，到屏幕边缘**不自动靠边**（widget 习惯）| – |
| 状态变更 | 1.5s 临时气泡（L2），8 动物动效切换（L1）| L1 + L2 |

**核心原则**：**任何交互都不会让牧场跳出一个新窗口**（控制舱除外）。所有信息都在场景内呈现。
