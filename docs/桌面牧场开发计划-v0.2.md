## 0. 本计划使用约定

### 0.1 工作量单位

- **小时**：单人写代码 / 调试
- **半天**：≈ 4 小时
- **工作日**：≈ 6 有效小时（不含日常 review / 会议）

### 0.2 准入 / 产出 / 不通过 / 验收 四个字段强制

每个 M 必须四字段齐全；**任一字段缺失视为本计划不完整，进度不更新**。

### 5.3 实施步骤

1. `useDockAndDrag`：mousedown 启动 drag → mousemove 计算 setBounds 同步 → mouseup 检测到边缘 1s 吸附
2. 拖拽边界事件：`ranch:set-bounds` IPC 写 prefs
3. `ContextMenu` 实现两条触达：
   - DOM 模式：`onContextMenu` 阻止默认，渲染 React 浮层菜单
   - 系统菜单模式：发 `ranch:right-click` 给主进程，主进程弹 Menu.buildFromTemplate
4. 默认走**系统菜单**（更原生，与现有控制舱托盘风格统一）
5. 双击动物 → 本地 event → `ranch:open-cockpit` IPC
6. 主进程维护"未读计数"：ranchWindow 隐藏或 sleeping 时，`agents:snapshot-changed` 触发 +1；窗口 title 改 `(N)`
7. 控制舱 `app.on('browser-window-focus')` 或显示事件触发 + IPC `ranch:reset-unread`

## 8. 风险与回退

| 风险 | 触发场景 | 回退 |
| --- | --- | --- |
| 视觉 A 档被吐槽"不到位" | 用户 screenshot 时反馈 | v0.3 切 C 档 SVG，**不影响数据层 / IPC** |
| `connectors.json` 默认 disabled 引发"为啥不下发真 agent" | 用户 ask | M5 内 add doc 注释，说明策略 |
| 牧场窗口过快导致掉帧 | i5/8GB 机器掉帧 | 减少 `@keyframes` 复杂度，将 motion 退到 `requestAnimationFrame` |
| 系统通知被 Win11 Focus 助手当掉 | 用户用勿扰模式 | 在牧场通知偏好里独立关 |
| FR-005 双击误触 | 用户连击 | 加 200ms 防抖 |
| 牧场被遮盖无法回收通知 | 用户切到桌面又回来 | 控制舱未读红点（FR-010）+ 反向 IPC |
| Q1 改 B 档导致 M1+M2 翻倍 | 用户拍 B | 跳过 M1+M2 直接进 M1-Visual 改造，但 FR-001~FR-011 不变 |
| desktop mode 透明穿透在 Win11 DWM 下行为异常 | 鼠标点圈外**没有**穿透到下面应用 | M0 在 Win11 / macOS 两套系统下 smoke 验证；若失败回退到 floating-only 模式（Q9 默认改回 floating）|
| desktop mode 在 Win11 多 virtual desktop 间不同步 | 切桌面后牧场不显示 | v0.2 用 `setVisibleOnAllWorkspaces(true, { visibleOnFullScreen: true })` 缓解；v0.3 加 MonitorSwitch 监听 |
| 中央 8 卡卖点改坏 | 桌面牧场改造时无意改动卖点区（`NiuMaAvatar` / `is-hot` keyframe / 4×2 grid） | **§6.6 卖点保护硬约束**：所有改动提交前必须 `git diff` 卖点区文件为空；任何 PR 触动卖点文件 = 必须先写"卖点保护说明 + 用户明示同意" |
