# 🐄 多Agent牛马看板主页面 UI/UX 深度重构需求

> **⚠️ 本方案已被 [`docs/主页面重构方案-v3.0-控制舱收口.md`](./主页面重构方案-v3.0-控制舱收口.md) 收口，本文档仅供历史参考。**
> **收口日期**：2026-07-03
> **收口原因**：本方案的"重装饰路线"（赛博网格 + 霓虹投影 + 工位装饰物 + 拖拽派活 + 抛物线投喂）与 §〇·quinary 简洁规范 + §〇·quarter 卖点保护有冲突，v3.0 收口时全部挪到 P2 暂缓（v0.4+ 评估）。

为了进一步提升看板的视觉冲击力、操作趣味性以及信息传达效率，针对主页面（Cockpit Workspace）提出以下 UI/UX 深度重构需求：

---

## 一、 视觉层重构 (Visual Polish)

### 1. 赛博格子间 (Cyber-Cubicle) 舞台化
*   **现状**：目前的 `avatar-stage` 只是一个带边框的深色背景方块。
*   **重构需求**：
    *   **背景网格化**：格子间背景增加微弱的 `10px * 10px` 赛博网格线（Grid Pattern），使用 `background-image: linear-gradient` 实现。
    *   **状态霓虹投影**：当 Agent 处于忙碌或警报状态时，格子间底部产生对应状态色（如红色、黄色）的**霓虹呼吸投影**（`box-shadow` 呼吸动画）。
    *   **工位装饰物**：根据 Agent 的不同性格，在格子间角落绘制微缩的装饰物（如 Codex 的工位放个“排障扳手”，Qoder 的工位放个“木鱼”，Hermes 的工位放个“躺椅”）。

### 2. 状态管道 (Data Pipeline) 实体化与流光
*   **现状**：左侧的 Connector 管控区与中央的牛马工位矩阵在视觉上是割裂的。
*   **重构需求**：
    *   **SVG 连线**：在左侧 Connector 卡片与中央 Agent 卡片之间，使用 SVG 绘制动态连接线。
    *   **流光动画**：当 Connector 处于 `ready` 且 Agent 正在执行任务时，连接线上有发光的粒子（Dash Array Animation）从左向右流动，代表“状态数据正在实时打通”。

---

## 二、 交互体验重构 (UX & Micro-interactions)

### 1. 拖拽派活 (Drag & Drop Dispatch)
*   **现状**：派活需要先选中 Agent，然后在右侧面板输入或点击快捷任务。
*   **重构需求**：
    *   **任务卡片化**：将右侧的“快捷任务”重构为可拖拽的“任务卡片”。
    *   **拖拽即拉磨**：用户可以直接按住一个任务卡片，拖拽到中央矩阵的任意一个牛马格子间上释放，瞬间触发该 Agent 的“拉磨”动画。

### 2. 趣味“物理反馈” (Physics Feedback)
*   **现状**：点击动作按钮（如画饼、咖啡）只是瞬间切态，缺乏物理动感。
*   **重构需求**：
    *   **抛物线投喂**：点击“给杯咖啡”或“画个大饼”时，对应的 ☕ 或 🥧 Emoji 从按钮位置以**抛物线轨迹**飞向当前选中的牛马格子间，落入格子间时触发轻微的“震动”和“溅起粒子”效果。
    *   **鼠标悬停“戳一戳”**：鼠标悬停在牛马 Emoji 上时，Emoji 产生受惊缩放，并随机弹出一句吐槽气泡（如“别戳了，再戳宕机了！”）。

---

## 三、 布局与响应式重构 (Layout & Responsiveness)

### 1. 右侧控制台“抽屉化” (Drawer Console)
*   **现状**：右侧控制台常驻，占用 400px 宽度，导致中等分辨率下牛马矩阵被严重挤压。
*   **重构需求**：
    *   **默认收起**：右侧控制台默认收起为一条极窄的“状态指示边栏”。
    *   **点击滑出**：点击任意 Agent 卡片时，控制台以平滑的 `spring` 动画向左滑出。
    *   **遮罩层**：在小屏下，滑出的控制台带有半透明遮罩，点击遮罩自动收起。

### 2. 终端日志“全屏/折叠”切换 (Terminal Toggle)
*   **现状**：终端日志高度固定在 150px，查看长日志时非常痛苦。
*   **重构需求**：
    *   **高度可拉伸**：终端面板顶部增加一条拖拽手柄，允许用户上下拉伸调整高度。
    *   **一键全屏**：终端右上角增加“全屏/折叠”按钮，点击后终端面板向上展开，覆盖主工作区，方便排查复杂 Bug。
